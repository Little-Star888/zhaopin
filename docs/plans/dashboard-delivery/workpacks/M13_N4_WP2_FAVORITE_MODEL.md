# 工作包 M13-N4-WP2：收藏模型重构

> 目标：将"待投递"统一替换为"收藏"交互模型
> 角色：前端、后端
> 预估改动量：~60行JS + 20行CSS + 后端接口确认/新增

## 1. 前置条件
- M13-N2 通过（首页批次视图就绪，收藏状态联动可在首页验证）

## 2. 读文件
| 文件 | 读取目的 |
|------|---------|
| `crawler/extension/dashboard.js` | 岗位卡片渲染、收藏/投递状态管理 |
| `crawler/extension/dashboard.css` | 卡片样式、按钮样式、五角星图标 |
| `crawler/extension/dashboard.html` | 卡片模板结构 |
| `controller/db.js` | 确认 `scraped_jobs` 表是否有收藏状态字段 |
| `controller/jobs-handler.js` | 确认是否有更新收藏状态的 API |

## 3. 改动规格
- **文案替换**：
  - 全局搜索"待投递""已投递""取消选择"，替换为"收藏""取消收藏"
  - 按钮文案统一为"收藏"（未收藏）/ "取消收藏"（已收藏）
- **图标规范**：
  - 未收藏：黄色五角星（★）+ "收藏"文字
  - 已收藏：灰色五角星（☆）+ "取消收藏"文字
  - 使用 CSS class 切换状态，不硬编码内联样式
- **首页交互**：
  - 卡片操作区域显示黄色五角星 + "收藏"按钮
  - 点击收藏后，五角星变灰色，按钮消失或变为"取消收藏"
- **工作台详情交互**：
  - 详情页显示"取消收藏"按钮
- **联动更新**（取消收藏时）：
  - 关闭当前 50/50 分屏
  - 从工作台列表移除该岗位
  - 首页对应卡片五角星恢复为未收藏状态
- **后端确认/新增**（必须先完成再改前端）：
  - 检查 `scraped_jobs` 表是否已有 `is_favorite`（或类似）字段
  - 如无：ALTER TABLE 新增 `is_favorite INTEGER DEFAULT 0`
  - 确认/新增 `/api/jobs/:id/favorite` 接口（POST 切换收藏状态）
  - 收藏/取消收藏必须落库，确保刷新页面后状态不丢失
