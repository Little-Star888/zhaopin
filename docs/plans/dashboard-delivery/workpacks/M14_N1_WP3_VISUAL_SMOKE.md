# 工作包 M14-N1-WP3：视觉回归冒烟检测

> 目标：端到端验证工具栏按钮视觉与状态一致性
> 角色：测试
> 预估改动量：无代码改动

## 1. 前置条件
- M14-N1-WP1 ~ WP2 全部通过

## 2. 测试场景

### 场景 1：按钮尺寸一致性
1. 打开 Dashboard，进入工作台
2. 用 DevTools 选中 `.res-bar` 内每个按钮
3. 记录每个按钮的 `offsetHeight` 和 `offsetWidth`
4. 所有值应完全相同（误差 0px）
5. 打开 50/50 分屏，重复以上验证

### 场景 2：首屏唯一红色
1. 刷新 Dashboard，进入工作台
2. 确认仅"编辑"按钮为红色
3. 确认"AI优化"按钮为黑色/深色
4. 确认"查看"、"保存"、"AI配置"、"下载"均非红色

### 场景 3：Tab 互斥切换
1. 点击"查看"→ 确认"查看"变红，"编辑"变黄
2. 点击"编辑"→ 确认"编辑"变红，"查看"变黄
3. 反复切换 3 次，确认无异常

### 场景 4：动作按钮不持久变红
1. 点击"保存"→ 确认短暂反馈后恢复，不持久变红
2. 点击"AI配置"→ 确认配置面板切换，按钮不持久变红
3. 点击"下载"→ 确认下载触发，按钮不持久变红

### 场景 5：AI 按钮 hover/disabled 态
1. hover "AI优化"按钮 → 确认背景变红、边框变红
2. 未配置 AI 时 → 确认"AI优化"灰色底 + 灰色边框（无"灰底黄边"）
3. hover disabled 的"AI优化"→ 确认无视觉变化

### 场景 6：focus-visible 焦点样式
1. 用 Tab 键依次导航到每个按钮
2. 确认焦点轮廓清晰可见，不被 border 遮盖
3. 焦点轮廓样式统一

## 3. 验证清单
- [ ] 所有场景通过
- [ ] 默认工具栏与分屏工具栏表现一致
- [ ] 不存在 JS 交互回归（view/edit 切换、保存、AI 优化正常工作）
- [ ] 截图留存修复前后对比（首屏默认态、hover 态、disabled 态）
