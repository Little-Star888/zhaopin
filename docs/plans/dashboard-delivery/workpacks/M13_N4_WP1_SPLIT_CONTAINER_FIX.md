# 工作包 M13-N4-WP1：50/50 分屏容器限定

> 目标：分屏只在工作台页面内显示，不污染首页
> 角色：前端
> 预估改动量：~30行CSS + 20行JS

## 1. 前置条件
- M13-N2 通过（首页批次视图就绪）

## 2. 读文件
| 文件 | 读取目的 |
|------|---------|
| `crawler/extension/dashboard.html` | 页面结构、容器嵌套关系 |
| `crawler/extension/dashboard.css` | 分屏样式定义 |
| `crawler/extension/dashboard.js` | 分屏打开/关闭逻辑、路由切换 |

## 3. 改动规格
- **结构限定**：
  - 确认 `splitView` 容器位于工作台页面（`#resume`）内部
  - 从首页（`#home`）移除任何分屏相关的 DOM 或引用
- **定位方案**（二选一）：
  - 方案 A（推荐）：`position: absolute` + `inset: 0`，相对于工作台容器定位
  - 方案 B：同级 flex 切换，工作台列表和分屏容器互斥显示
- **交互逻辑**：
  - 点击工作台岗位卡片：隐藏工作台列表 → 显示分屏
  - 关闭分屏：隐藏分屏 → 恢复工作台列表
  - 分屏不参与文档流，不撑长页面
- **首页隔离**：
  - `#home` 的 hashchange 事件不触发分屏
  - 首页卡片点击不打开分屏（首页只有收藏操作）

- [ ] 分屏打开时页面不被撑长，无需滚动
- [ ] 首页不出现分屏内容
- [ ] 关闭分屏后工作台列表恢复正常
- [ ] 首页和工作台之间来回切换，分屏状态正确
