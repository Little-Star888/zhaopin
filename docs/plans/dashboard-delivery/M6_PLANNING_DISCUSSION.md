## 1. 用户需求

- 插件 Popup UI 色调替换为此次定下的 6 个 PANTONE 色
- 文字色不限于当前颜色（比 Dashboard 灵活）
- 单独开一个里程碑（M6）

## 3. 分歧记录

### 分歧 1（已解决）：是否实现主题切换功能？

| 立场 | 顾问 | 专家 | 我方 |
|------|------|------|------|
| 主张 | 实现 `data-theme` 属性切换 | 只做静态换色 | 只做静态换色 |

**专家拍板**：
> "现在是收口，不是做配置系统。主题切换器会把简单视觉需求做成状态管理需求。"

**共识**：只做静态换色，不做主题切换器。

---

### 分歧 2（已解决）：是否提取内联样式为独立 CSS 文件？

| 立场 | 顾问 | 专家 | 我方 |
|------|------|------|------|
| 主张 | 必须提取为 popup.css | 建议提取，合理最小重构 | 同意提取 |

**专家拍板**：
> "建议把 popup.html 内联样式提成独立 popup.css。这是合理的最小重构，而且能把 popup 颜色体系和 Dashboard 彻底分离。"

**共识**：提取内联样式为独立 `popup.css`。

---

### 不确定项（已解决）：哪个 PANTONE 色替代绿色作为 Popup 主色？

**专家拍板**：
> "替代绿色主色，我建议优先 Orange Rust。理由：popup 是操作入口，需要清晰主按钮色；黄色太轻，Aquamarine 太接近旧'工具感/成功态'语义，Rust 更稳、更有主操作感。"

**共识**：Orange Rust (#C75B4A) 作为 Popup 主色。

---

## 4. 专家最终方案

### 4.1 M6 范围

- 静态换色（不做主题切换器）
- 提取内联样式为独立 popup.css
- Orange Rust 作为主色
- 轻量级 Neumorphism（不用 Glassmorphism）
- 文字色独立，优先可读性

### 4.2 颜色映射

| 用途 | 新色值 | 说明 |
|------|--------|------|
| 页面背景 | Citron `#E6E6A1` | 与 Dashboard 一致 |
| 主色（按钮、Toggle、进度条） | Orange Rust `#C75B4A` | 替代绿色 |
| 成功状态 | Aquamarine `#A8D0E6` | 替代绿色成功 |
| 警告状态 | Radiant Yellow `#F9A825` | 替代橙色 |
| Header 渐变 | Orange Rust → Pink Dogwood | 替代绿色渐变 |
| 正文 | `#333333` | 独立文字色 |
| 次要文字 | `#666666` | 独立文字色 |
| 卡片背景 | `white` | 保持 |

### 4.3 Neumorphism 参数

popup 使用轻量级 Neumorphism（减小阴影）：

```css
.popup-section {
    background: var(--c-citron);
    border-radius: 12px;
    box-shadow:
        4px 4px 8px rgba(199, 91, 74, 0.12),
        -4px -4px 8px rgba(253, 231, 152, 0.5);
}
```

### 4.4 结构建议

- M4：Dashboard 主界面重构
- M5：多平台纯文档设计（可并行）
- M6：插件 Popup/UI 色调与样式重构
