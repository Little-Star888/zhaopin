# 架构说明

## 组件

### `controller/`

本地 HTTP 控制面，负责：

- 接收任务
- 管理本地队列与状态
- 生成批量运行摘要
- 管理飞书投递目标

### `crawler/extension/`

Chrome 扩展，负责：

- 在 Boss 直聘页面执行采集
- 与本地控制面联动
- 回传任务执行结果

### `tests/`

保留结构验收脚本，用于校验控制面接口和基础行为。

## 数据流

1. 用户或脚本向控制面提交任务
2. 扩展从控制面读取待执行任务
3. 扩展完成采集后向控制面回报结果
4. 控制面按配置决定是否执行投递

## 默认安全策略

- 默认不开启真实投递
- 默认不附带任何真实凭证
- 默认仅提供示例批次和示例配置
